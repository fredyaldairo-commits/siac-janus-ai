Modelo base · interpretable

1. Regresión Logística

Combinación lineal de las variables pasada por una sigmoide. Es la base: define el peso (βᵢ) y el signo esperado de cada variable.
P(aprobar | x) = σ(β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ)
σ(z) = 1 / (1 + e−z)
Se ajusta por máxima verosimilitud: ℓ(β) = Σ [ yᵢ·log(pᵢ) + (1−yᵢ)·log(1−pᵢ) ]. Cada βᵢ es el peso; su signo dice si la variable sube (+) o baja (−) la aprobación.

Boosting por gradiente

2. XGBoost — gradiente y hessiano

Suma árboles que corrigen, uno a uno, el error del anterior. Usa la 1ª derivada (gradiente) y la 2ª (hessiano) para decidir rápido y bien.
ŷ = Σk fk(x)  ·  árboles aditivos
Obj = Σᵢ L(yᵢ, ŷᵢ) + Σk Ω(fk)  ·  (Ω = penalización por complejidad)
peso óptimo de hoja: w* = − G / (H + λ)  ·  G=Σ gradiente, H=Σ hessiano
El gradiente indica la dirección del error; el hessiano su curvatura → el paso es más certero (converge más rápido). Va segundo, tras la regresión, aportando lo no-lineal.
Bagging · anti-sobreajuste

3. Random Forest

Muchos árboles entrenados sobre muestras distintas; se promedian. El promedio reduce la varianza y evita el sobreajuste de un solo árbol.
P = (1/B) Σb=1B Tb(x)  ·  promedio de B árboles
Al promediar errores independientes, el aprendizaje queda lo más cercano y robusto posible. Va tercero, estabilizando antes de la red.

Red neuronal · GELU + Sigmoide

4. Red Neuronal

Capas que combinan todo lo anterior de forma no-lineal. Ocultas con GELU (rápida por el gradiente y sin neuronas muertas); salida con sigmoide (probabilidad continua, no 0/1 duro).
a(l) = GELU( W(l) a(l−1) + b(l) )  ·  capas 31 → 64 → 32 → 1
GELU(z) = 0.5·z·(1 + tanh[ √(2/π)·( z + 0.044715·z³ ) ])
salida: σ(z) = 1/(1+e−z)  ·  pérdida: BCE  ·  optimizador: Adam
Cada neurona tiene pesos W y sesgo b: una variable más significativa recibe mayor peso. Entrena por descenso de gradiente minimizando la pérdida. GELU es suave (derivable en todo punto), así el gradiente fluye mejor y —a diferencia de ReLU— no “apaga” neuronas a 0.
